Introduction

The NERDm schema is used to encode metadata describing various kinds of resources--datasets, data collections, web portals, software, and tools--that are offered by NIST. Resources are the main kind of "thing" that NERDm metadata is used to describe, and there are different types resources.

NERDm metadata is typically encoded in JSON format, but this guide attempts to describe the metadata without relying on knowledge of that format. In particular, we describe the metadata schema in terms of a few concepts:

property
Metadata describe things like resources in terms of a set of properties. A property is a piece of metadata that has a name and a value. A value can take the form of a simple value like text, a number, or true/false. It can a also be list of values of those simple types, or it can be a set of "sub-" properties, each with their own name and value.

value type
The value type indicates what kind value it has, like text (i.e. a string), a number (e.g. integer or decimal), or a boolean (true or false). The value type can refer to a set of specific sub-properties with prescribed names. We can give that set of prescribed properties a name, making it a Named Type.

As an example, a property providing contact information is called contactPoint. Its value is a set of sub-properties that includes fn (full name), hasEmail (an email address), phoneNumber, etc. The set of subproperties that is collectively the contact information is a type called ContactInfo.

Resource Types

Named Type
Resource

What it represents:
A dataset, collection, service, tool or other digital resource that enables data-driven research.

How it is used:
Resource is a type of metadata object that can occur as a value to an @type property.

Properties:

Property of the Resource type
title
What it represents:
A human-readable, descriptive name of the resource
Value type:
text
Example Values:
"The NIST High Temperature Superconducting Materials Database",
"Organic Contaminants in Non-Fortified Human Milk"
Property of the Resource type
description
What it represents:
Human-readable description (e.g., an abstract) of the resource
Value type:
text
Property of the Resource type
accessLevel
What it represents:
The degree to which this dataset could be made publicly-available, regardless of whether it has been made available
Value type:
text
Allowed Values:
public
Data asset is or could be made publicly available to all without restrictions
restricted public
Data asset is available under certain use restrictions
private
Data asset is not available to members of the public
Property of the Resource type
@type
What it represents:
a list of type names that indicate the kind of resource being described.
Value type:
list of type names
Example Values:
"nrdp:DataPublication", "nrdp:SRD"
Notes:
  • A resource can be catagorized as being of several types simultaneously
  • The first type listed is considered the primary type

Glossary

property
a piece of metadata describing something composed of a name and a value

named type
a special value type that is given a name. Named types are most often a "complex type"--sets of sub-properties, like ContactInfo or Resource, but sometimes they can refer a text value type that has a specific format, like ISO8601DateRange.

resource
A digital object or tool--such as a dataset, data collection, web portal, database, or software--that can enable data-driven science.

type (or value type)
a category of value that a metadata property can have. Different types include text, a number, a list of values, or a set of properties.
